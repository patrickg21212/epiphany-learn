## Content Notes

This is the **action module** — everything before this was learning, this is doing. The tone throughout:
- Practical and direct — every section produces something the user can apply
- Assumes the reader has finished modules 1-5 and is ready to act
- Honest about failure modes — doesn't sugarcoat the difficulty of change management
- Ends with a concrete, fill-in-the-blank action plan the user takes with them

## Lesson Summaries

### Lesson 1: Picking Your First AI Project
The four-filter framework for evaluating first use cases (reversible errors, measurable baseline, repetitive task, visible win). Side-by-side table of high-win vs. worst-first use cases. Four mistake patterns to avoid when scoping a first project. Uses `RevealCard` for decision filters and avoidance patterns, `Quiz` with 3 questions.

### Lesson 2: Setting Expectations
The two fears driving resistance (job replacement and more work). Five-part kickoff conversation framework. Four expectation traps with reveals. Specific strategy for the skeptic — make them a quality reviewer. Uses `RevealCard` for fears and traps, `Quiz` with 3 questions.

### Lesson 3: Measuring Success
Three core metrics: time saved, outcomes captured, error rate. Baseline setup protocol (5 steps, one week before launch). 2-week evaluation window with go/revisit/stop thresholds. Healthy early-stage benchmarks for 3 common use cases. Uses `RevealCard` for metric definitions, comparison table, `Quiz` with 3 questions.

### Lesson 4: First-Project Mistakes and Your Action Plan
Six failure modes with reveals. Full 8-step action plan template the user fills in. Each step includes time estimate and specific output. Evaluation framework at the 2-week mark (3-metric scorecard). Ends with `ModuleComplete` — final lesson in the course. Uses `RevealCard` for mistake patterns, template fill-in, `Quiz` with 3 questions.
